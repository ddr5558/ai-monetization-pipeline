## 1단계: 실제 데이터 준비

**[필수]** 이 가이드는 실제 회사 데이터로 진행합니다. 샘플/더미 데이터는 사용하지 마세요.

| 대상 | 실습에서 사용한 값 | 회사 적용 방법 |
|------|-------------------|--------------|
| 블로그 주제 | 재테크 노하우, 번아웃 극복 | 회사의 실제 주력 제품이나 브랜드 키워드를 blogGenerator.js의 프롬프트 변수에 반영합니다. |
| 챗봇 응대 로직 | 배송 조회 | server.js의 /webhook 라우트에 '교환/반품', '영업시간 안내' 등 회사에 필요한 인텐트를 추가합니다. |
| 이미지 스타일 | 프리미엄 골드 럭셔리 | 브랜드 가이드라인에 맞는 스타일 키워드를 프롬프트 생성 함수에 고정값으로 설정합니다. |

## 2단계: 데이터 교체

정적인 샘플 텍스트 대신 실제 비즈니스 데이터를 연결하여 자동화 수준을 높입니다.

**지원 형식**: JSON, MySQL/PostgreSQL (DB), Excel/CSV

학습자가 알려준 데이터에 맞게 아래 절차를 함께 진행하세요:
코드 내에 하드코딩된 '배송 준비 중'이라는 메시지 대신, 회사의 쇼핑몰 DB 또는 운송장 API에서 실제 상태를 조회하도록 코드를 수정합니다.

**[STOP]** 데이터 교체가 완료될 때까지 함께 진행하세요.

## 3단계: 결과 검증

교체한 데이터로 실행해봅시다:
curl 명령어를 사용하여 실제 주문 번호를 보냈을 때 DB에 저장된 실제 배송 상태가 출력되는지 확인합니다.

**검증 기준:**
- 에러 없이 실행 완료
- 출력 결과에 실제 데이터가 반영됨 (샘플 데이터 흔적 없음)
- 결과 파일이 정상 생성됨

에러가 발생하면 함께 디버깅하세요.

**[STOP]** 위 3가지 검증 기준을 모두 통과할 때까지 함께 진행하세요.

## 4단계: 정기 실행 (선택)

"이걸 자동으로 돌아가게 설정할 수 있어요. 해볼까요?"

**방법**: GitHub Actions
**권장 주기**: 블로그 생성은 주 3회, 챗봇 서버는 24시간 상시 가동
.github/workflows 폴더 내에 yml 파일을 생성하여 매일 아침 정해진 시간에 스크립트가 실행되도록 설정합니다.

```
on:
  schedule:
    - cron: '0 0 * * *' # 매일 자정 실행
```

**[STOP]** 설정 완료 또는 건너뛰기를 기다리세요.
